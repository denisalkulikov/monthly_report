# 📊 Ежемесячный отчёт

Приложение на базе **NiceGUI** для автоматического формирования ежемесячных отчётов на основе данных из PostgreSQL. Позволяет заполнять Excel-шаблоны данными из базы и копировать значения из другого файла.

---

## 🚀 Возможности

- 📅 Выбор года и месяца для отчёта
- 💰 Ввод суммы реализации (отгрузки)
- 🔍 Получение данных из PostgreSQL:
  - Таблица `kamtent.monthly_group_product` (суммы по направлениям и группам продуктов)
  - Таблица `kamtent.sales` (данные по оплатам и отгрузкам)
- 📂 Загрузка двух Excel-файлов (`.xlsm`):
  - **Файл-шаблон** — в него записываются данные из БД
  - **Файл-источник** — из него копируются значения в шаблон
- 📝 Автоматическое заполнение листов:
  - `служ` — месяц и год
  - `тит` — год
  - `общ` — суммы по направлениям (ОАИ, ТК, АНГАРЫ, РЕКЛАМА, КН)
  - `сегм` — отгрузки по направлениям и расчёт разницы
  - `Ателье` — суммы по РЕКЛАМА, ТК и итоговый расчёт
  - `Авто` — данные по ОАИ (розница, потребители, ответственность)
  - `ТК` — данные по направлениям ТК
  - `Реклама` — данные по рекламным продуктам
  - `Ангар` — данные по отгрузкам и продуктам
  - `Ком` — данные по КН (розница, потребители, ответственность)
- 📋 Копирование данных из файла-источника в результирующий файл
- 📊 Визуальный прогресс-бар при обработке
- 📥 Автоматическая загрузка готового отчёта

## 🛠️ Технологии

- **Python 3.12+**
- **NiceGUI** — веб-интерфейс
- **PostgreSQL** — база данных
- **psycopg2** — драйвер для работы с БД
- **openpyxl** — работа с Excel-файлами
- **uv** — менеджер пакетов

## 📦 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/denisalkulikov/monthly_report.git
cd monthly_report
```

### 2. Установка зависимостей через uv
```bash
uv sync
```

### 3. Настройка переменных окружения
Создайте файл `.env` в корне проекта:
```
DB_HOST=
DB_NAME=
DB_USER=
DB_PASSWORD=
DB_PORT=
```

## 📖 Инструкция по использованию

### 1. Настройка отчёта
- Выберите год и месяц
- Введите сумму реализации (отгрузки)
- Нажмите «Получить данные» — данные загрузятся из БД, и вы увидите раскрывающиеся списки с суммами по направлениям

### 2. Загрузка файлов
- Нажмите «Файл-шаблон (XLSM)» и выберите Excel-файл, в который будут записаны данные
- Нажмите «Файл-источник (XLSM)» и выберите файл, из которого будут копироваться значения

### 3. Создание отчёта
- Нажмите «Создать отчет»
- Дождитесь завершения обработки (появится прогресс-бар)
- Готовый файл автоматически скачается в папку Загрузки

## 📄 Требования к Excel-файлам
- Формат: .xlsm (с поддержкой макросов)
- Должны содержать листы с указанными названиями (служ, тит, общ, сегм, Ателье, Авто, ТК, Реклама, Ангар, Ком)
- Ячейки для вставки данных должны соответствовать заданным адресам (см. код)

## 🐛 Возможные проблемы и решения
|Проблема|Решение|
|---|-|
|Ошибка подключения к БД|Проверьте параметры в файле `.env`|
|Не найден лист в файле|Убедитесь, что Excel-файл содержит нужный лист с точным названием|
|Ошибка при копировании данных|Проверьте, что файл-источник содержит ячейки с данными|
|Не скачивается файл|Проверьте настройки браузера (разрешите автоматическую загрузку)|

---

## 📝 Примечания
- Приложение сохраняет обработанный файл во временную папку, а затем предлагает скачать его 
- Файл-источник не изменяется, данные копируются только в результирующий файл 
- Все временные файлы автоматически удаляются после завершения обработки
